## Webhook Overview

Webhooks enable real-time communication between webMOBI services and your applications. When specific events occur, webMOBI sends HTTP POST requests to your configured endpoints with event data.

## Setup Guide

<Steps>
  <Step title="Create Webhook Endpoint" icon="server">
    Develop an HTTP endpoint that can receive POST requests from webMOBI.
    
    ```javascript
const express = require('express');
const app = express();

app.use(express.json());

app.post('/webhooks/webmobi', (req, res) => {
  const event = req.headers['x-webmobi-event'];
  const signature = req.headers['x-webmobi-signature'];
  
  // Verify webhook signature
  if (!verifySignature(req.body, signature)) {
    return res.status(401).send('Invalid signature');
  }
  
  // Process the event
  console.log(`Received event: ${event}`, req.body);
  
  // Return 200 OK to acknowledge receipt
  res.status(200).send('OK');
});

app.listen(3000, () => console.log('Webhook server running'));
```
  </Step>
  <Step title="Secure Your Endpoint" icon="shield">
    Implement security measures to verify webhook authenticity.
    
    <Expandable title="Signature Verification" default-open="false">
      webMOBI signs all webhook requests using HMAC-SHA256.
      
      ```javascript
function verifySignature(payload, signature) {
  const crypto = require('crypto');
  const expectedSignature = crypto
    .createHmac('sha256', process.env.WEBMOBI_WEBHOOK_SECRET)
    .update(JSON.stringify(payload))
    .digest('hex');
  
  return crypto.timingSafeEqual(
    Buffer.from(signature, 'hex'),
    Buffer.from(expectedSignature, 'hex')
  );
}
```
      </Expandable>
    
    <Callout kind="alert">Always verify webhook signatures to prevent spoofing attacks.</Callout>
  </Step>
  <Step title="Register Webhook" icon="plus">
    Configure your webhook URL in the webMOBI dashboard.
    
    <CodeGroup tabs="REST API,Dashboard">
      ```bash
curl -X POST https://api.webmobi.com/v1/webhooks \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-app.com/webhooks/webmobi",
    "events": ["event.created", "attendee.registered"],
    "secret": "your_webhook_secret"
  }'
```
      ```javascript
// Dashboard configuration
const webhookConfig = {
  url: 'https://your-app.com/webhooks/webmobi',
  events: ['event.created', 'attendee.registered', 'event.updated'],
  active: true,
  retryPolicy: {
    maxRetries: 3,
    backoffMultiplier: 2
  }
};
```
    </CodeGroup>
  </Step>
  <Step title="Test Webhook" icon="check-circle">
    Send test events to verify your endpoint is working correctly.
    
    ```bash
curl -X POST https://api.webmobi.com/v1/webhooks/test \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -d '{"event":"event.created","webhook_id":"wh_12345"}'
```
  </Step>
</Steps>

## Payload Structure

<ExpandableGroup>
  <Expandable title="Common Payload Format" default-open="true">
    All webhook payloads follow a consistent JSON structure.
    
    ```json
{
  "id": "evt_1234567890",
  "object": "event",
  "type": "event.created",
  "created": 1640995200,
  "data": {
    "object": {
      "id": "event_123",
      "name": "Annual Conference 2024",
      "start_date": "2024-01-15T09:00:00Z",
      "end_date": "2024-01-17T17:00:00Z",
      "venue": "Convention Center",
      "capacity": 500,
      "status": "active"
    }
  },
  "metadata": {
    "source": "webmobi_dashboard",
    "user_id": "user_456",
    "organization_id": "org_789"
  }
}
```
    
    | Field | Type | Description |
    |-------|------|-------------|
    | `id` | string | Unique event identifier |
    | `object` | string | Object type (`event`, `attendee`, etc.) |
    | `type` | string | Event type (e.g., `event.created`) |
    | `created` | integer | Unix timestamp of event creation |
    | `data` | object | Event-specific data payload |
    | `metadata` | object | Additional context information |
  </Expandable>

## Error Handling and Retries

<Callout kind="info">webMOBI implements automatic retry logic for failed webhook deliveries.</Callout>

| HTTP Status | Action | Retry Behavior |
|-------------|--------|----------------|
| 2xx | Success | No retry |
| 3xx | Redirect | Follow redirect, retry if fails |
| 4xx | Client Error | No retry (permanent failure) |
| 5xx | Server Error | Retry with exponential backoff |

<Expandable title="Retry Configuration" default-open="false">
  Customize retry behavior for your webhooks.
  
  ```javascript
const retryConfig = {
  maxRetries: 5,
  initialDelay: 1000, // 1 second
  maxDelay: 300000,   // 5 minutes
  backoffMultiplier: 2,
  retryableStatuses: [408, 429, 500, 502, 503, 504]
};
```
</Expandable>
